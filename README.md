# 2. Обнаруженные уязвимости (три примера)
Уязвимость 1: vsftpd 2.3.4 - Backdoor Command Execution
Описание: В версии FTP-сервера vsftpd 2.3.4 был обнаружен бэкдор. При отправке строки, содержащей последовательность :) в имени пользователя, на порту 6200 открывается командная оболочка с правами root.

Ссылка на Exploit-DB: https://www.exploit-db.com/exploits/17491

EDB-ID: 17491

Уязвимость 2: UnrealIRCd 3.2.8.1 - Backdoor Command Execution
Описание: В IRC-сервере UnrealIRCd 3.2.8.1 обнаружен бэкдор. При отправке команды, содержащей системный вызов (например, AB; <команда>), сервер выполняет её в системе с правами пользователя, под которым запущен IRC-сервер.

Ссылка на Exploit-DB: https://www.exploit-db.com/exploits/16922

EDB-ID: 16922

Уязвимость 3: Samba 3.X - Usermap Script (CVE-2007-2447)
Описание: В Samba версий с 3.0.20 по 3.0.25rc3 существует уязвимость удаленного выполнения кода. Злоумышленник может передать команду через параметр username map script, что позволяет выполнить произвольные команды с правами суперпользователя.

Ссылка на Exploit-DB: https://www.exploit-db.com/exploits/16320

EDB-ID: 16320

# Доработки:

1. Подготовка окружения
 
Целевая машина: Metasploitable (IP: 192.168.186.129)

Атакующая машина: Ubuntu Server (IP: 192.168.186.128)

Установлен TShark для захвата трафика в консольном режиме

2. Проведение сканирований и захват трафика

Для каждого типа сканирования выполнялась последовательность:

Запуск TShark для захвата трафика на интерфейсе ens33 с фильтром по IP цели

Выполнение соответствующей команды Nmap

Остановка захвата и сохранение файла .pcapng

Тип сканирования	Команда Nmap	Файл захвата

SYN	sudo nmap -sS 192.168.186.129	syn_scan.pcapng

FIN	sudo nmap -sF 192.168.186.129	fin_scan.pcapng

Xmas	sudo nmap -sX 192.168.186.129	xmas_scan.pcapng

UDP	sudo nmap -sU 192.168.186.129 --top-ports 20	udp_scan.pcapng

Результаты анализа трафика

1. SYN-сканирование (-sS)
   
Характеристики трафика:

Отправляются TCP-пакеты только с флагом SYN

На открытые порты сервер отвечает SYN/ACK

Сканер немедленно отправляет RST, разрывая соединение до завершения handshake'а

На закрытые порты сервер отвечает RST/ACK

Фрагмент захвата:


3 192.168.186.128 → 192.168.186.129 TCP 58 40175 → 139 [SYN]          # Запрос на открытый порт

13 192.168.186.129 → 192.168.186.128 TCP 60 139 → 40175 [SYN, ACK]   # Ответ сервера

14 192.168.186.128 → 192.168.186.129 TCP 54 40175 → 139 [RST]         # Сброс соединения

4 192.168.186.128 → 192.168.186.129 TCP 58 40175 → 8080 [SYN]         # Запрос на закрытый порт
15 192.168.186.129 → 192.168.186.128 TCP 60 8080 → 40175 [RST, ACK]   # Немедленный сброс

2. FIN-сканирование (-sF)

Характеристики трафика:

Отправляются TCP-пакеты только с флагом FIN

Открытые порты не отвечают (игнорируют пакет согласно RFC)

Закрытые порты отвечают RST/ACK

Фрагмент захвата:

text
3 192.168.186.128 → 192.168.186.129 TCP 54 61778 → 1723 [FIN]         # Запрос на закрытый порт

12 192.168.186.129 → 192.168.186.128 TCP 60 1723 → 61778 [RST, ACK]   # Ответ RST/ACK

Открытые порты (80, 21, 22, 23 и др.) в захвате не имеют ответных пакетов.

3. Xmas-сканирование (-sX)
   
Характеристики трафика:

Отправляются TCP-пакеты с флагами FIN, URG, PSH одновременно ("рождественская елка")

Открытые порты не отвечают

Закрытые порты отвечают RST/ACK

Пакеты легко идентифицируются по множеству установленных флагов

Фрагмент захвата:


3 192.168.186.128 → 192.168.186.129 TCP 54 63451 → 111 [FIN, PSH, URG]   # Xmas-пакет на открытый порт

4 ... → 199 [FIN, PSH, URG]                                               # Xmas-пакет на закрытый порт

13 192.168.186.129 → 192.168.186.128 TCP 60 199 → 63451 [RST, ACK]       # Ответ от закрытого порта

4. UDP-сканирование (-sU)

Характеристики трафика:

Отправляются пустые UDP-пакеты на целевые порты

Открытые UDP-порты могут отвечать данными (если сервис активен)

Закрытые порты отвечают ICMP "Destination Unreachable (Port unreachable)" (тип 3, код 3)

Фрагмент захвата:

19 192.168.186.128 → 192.168.186.129 UDP 42 60287 → 631 Len=0           # UDP-запрос на закрытый порт

20 192.168.186.129 → 192.168.186.128 ICMP 164 Destination unreachable   # ICMP-ответ Port Unreachable

26 192.168.186.129 → 192.168.186.128 DNS 104 Standard query response    # Ответ от открытого DNS-сервера (порт 53)

Сравнительная таблица ответов сервера

Тип сканирования	Открытый порт	Закрытый порт	Особенности трафика

SYN	SYN/ACK → RST (от сканера)	RST/ACK	Полный цикл handshake с разрывом, легко обнаруживается

FIN	Нет ответа	RST/ACK	Только FIN-пакеты, тишина от открытых портов

Xmas	Нет ответа	RST/ACK	Пакеты с флагами FIN+URG+PSH, ответы только от закрытых портов

UDP	Нет ответа или данные сервиса	ICMP Port Unreachable	UDP-пакеты + ICMP-ответы, медленное сканирование


SYN-сканирование является наиболее информативным и быстрым, но легко обнаруживается системами обнаружения вторжений (IDS) из-за характерной последовательности SYN → SYN/ACK → RST.

FIN и Xmas-сканирования относятся к "скрытным" типам, так как не создают полных соединений. Они полезны для обхода простых фильтров, но не могут отличить открытые порты от фильтруемых (в обоих случаях ответа нет).

UDP-сканирование принципиально отличается от TCP-методов, используя ICMP-протокол для определения состояния портов. Оно значительно медленнее из-за таймаутов и особенностей UDP.

Анализ захваченного трафика полностью подтверждает теоретические основы работы различных типов сканирования Nmap:

TCP-сканирования используют флаги протокола для получения информации

Открытые и закрытые порты ведут себя в соответствии с RFC

Ответы сервера позволяют однозначно определить состояние порта
